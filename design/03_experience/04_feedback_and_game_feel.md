# 反馈与手感

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义增量游戏中“数字变大、系统解锁、奖励掉落”的反馈标准。

## 关键反馈点

- 制作菜品完成。
- 首次发现新菜谱。
- 勇者属性提升。
- 菜品让战斗预测发生明显变化。
- 餐厅升级。
- 技能树节点点亮。
- 袭击倒计时进入紧急阶段。
- 讨伐成功。
- 获得稀有食谱、稀有食材或战利品。

## 手感原则

- 高频成长反馈要快。
- 稀有奖励反馈要慢一点、重一点。
- 数字增长要清楚显示前后变化。
- 自动战斗可以短，但不能像黑箱结算。

## 烹饪反馈

烹饪反馈应短、快、清楚，重点展示“这道菜解决了什么问题”。

| 场景 | 反馈方式 |
| --- | --- |
| 普通制作完成 | 快速完成音效、菜品图标弹出、属性变化跳字。 |
| 首次发现新菜谱 | 稍慢一点的解锁演出，记录进菜谱。 |
| 给勇者吃下适配菜 | 显示职业适配文本，例如“战士获得额外攻击准备”。 |
| 命中 Boss 缺口 | 显示预测变化，例如“重击风险降低”或“出血收益提高”。 |
| 材料不足 | 直接提示缺哪个食材，以及获取方式。 |

示例反馈文本：

- “青菜肉卷完成：战士精力 0/3 -> 3/3，额外攻击 1 次。”
- “霜盐冷盘完成：法师命中弱冰，爆发提高。”
- “香草轻食盒完成：弓箭手可破坏部位并叠加出血。”
- “蘑菇盾锅完成：多段攻击风险降低。”
