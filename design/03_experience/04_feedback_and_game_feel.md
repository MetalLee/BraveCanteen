# 反馈与手感

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义增量游戏中“数字变大、系统解锁、奖励掉落”的反馈标准。

## 关键反馈点

- 制作菜品完成。
- 首次发现新菜谱。
- 勇者属性提升。
- 菜品让战斗预测发生明显变化。
- 餐厅升级。
- 技能树节点点亮。
- 袭击倒计时进入紧急阶段。
- 讨伐成功。
- 获得稀有食谱、稀有食材或战利品。

## 手感原则

- 高频成长反馈要快。
- 稀有奖励反馈要慢一点、重一点。
- 数字增长要清楚显示前后变化。
- 自动战斗可以短，但不能像黑箱结算。

## 烹饪反馈

烹饪反馈应短、快、清楚，重点展示“这道菜解决了什么问题”。

| 场景 | 反馈方式 |
| --- | --- |
| 普通制作完成 | 快速完成音效、菜品图标弹出、属性变化跳字。 |
| 首次发现新菜谱 | 稍慢一点的解锁演出，记录进菜谱。 |
| 给勇者吃下适配菜 | 显示职业适配文本，例如“战士获得额外攻击准备”。 |
| 命中 Boss 缺口 | 显示预测变化，例如“重击风险降低”或“出血收益提高”。 |
| 材料不足 | 直接提示缺哪个食材，以及获取方式。 |

示例反馈文本：

- “青菜肉卷完成：战士精力 0/3 -> 3/3，额外攻击 1 次。”
- “霜盐冷盘完成：法师命中弱冰，爆发提高。”
- “香草轻食盒完成：弓箭手可破坏部位并叠加出血。”
- “蘑菇盾锅完成：多段攻击风险降低。”

## 厨师演出动画

厨师演出应服务于配餐爽感，而不是变成拖慢节奏的强制动画。首版建议采用“普通快、组合重、稀有可跳过”的节奏。

### 演出层级

| 层级 | 触发场景 | 时长 | 表现 |
| --- | --- | --- | --- |
| 普通制作 | 单道普通菜完成 | 0.4 - 0.7 秒 | 厨师切菜 / 翻锅一下，菜品图标弹出。 |
| 适配成功 | 菜品给到推荐职业或命中 Boss 缺口 | 0.8 - 1.0 秒 | 勇者头像点亮，预测栏出现正向跳字。 |
| 牌型触发 | 形成对子、元素宴、蘑菇顺子等 | 1.0 - 1.5 秒 | 5 格餐桌连线，厨师做收尾动作，牌型名称出现。 |
| 稀有料理 | 首次做出稀有菜或高阶魔物料理 | 1.5 - 2.2 秒 | 特写锅光、菜品落盘、记录进菜谱。 |
| 永久成长 | 触发封顶永久属性成长 | 1.2 - 1.8 秒 | 勇者属性条上升，显示永久标记。 |

所有超过 1 秒的演出都应允许点击加速。重复触发同一普通菜时，自动使用短演出。

### 厨师动作语言

不同菜品类型应有不同动作，让玩家不用读字也能感到差异：

| 菜品类型 | 厨师动作 | 视觉反馈 |
| --- | --- | --- |
| 攻击菜 | 重刀切肉、猛火翻锅 | 红色火花、攻击图标跳出。 |
| 防御菜 | 慢炖、盖锅压实 | 厚重蒸汽、盾牌图标出现。 |
| 生命菜 | 加汤、盛满大碗 | 温暖光圈、生命条上浮。 |
| 精力菜 | 快速翻炒青菜 | 绿色速度线、行动图标亮起。 |
| 元素菜 | 撒元素调味、锅中变色 | 火 / 冰 / 雷元素短效。 |
| 出血准备菜 | 精准摆盘、刀尖一闪 | 弓箭手标记或弱点图标亮起。 |
| 游侠复合菜 | 多种食材依次入锅 | Boss 情报图标翻开。 |

### 牌型演出

牌型触发时，演出应把“我凑成了”表达清楚：

- 对子：两道菜同时弹起，桌面出现一条短连线。
- 双份龙虾：两盘龙虾被推到桌中央，攻击图标放大，若触发永久成长则显示“永久攻击 +1”。
- 蘑菇顺子：5 道菜依次亮起蘑菇标签，连成一圈后全队盾牌图标点亮。
- 元素宴：同元素菜依次燃起或结霜，法师头像获得克制光效。
- 五职套餐：五个职业头像依次点亮，最后核心勇者获得额外高亮。

牌型演出结束后，应立刻回到可操作状态，并保留一条简短结果文本，例如“蘑菇顺子：全队防御提高”。
