# 自动战斗系统

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义讨伐战斗的结算方式、可读性要求和后续扩展边界。

## 战斗定位

自动战斗用于兑现玩家前序经营成果。玩家不在战斗中频繁操作，而是在战斗前通过勇者培养和队伍选择影响结果。

## 首版公式方向

首版采用简单攻防数值对比：

```text
队伍攻击 = 所有出战勇者攻击之和
队伍魔法攻击 = 所有出战勇者魔法攻击之和
队伍生命 = 所有出战勇者生命之和
队伍防御 = 所有出战勇者防御之和
队伍技能强度 = 所有出战勇者技能强度之和

有效物理伤害 = max(1, 队伍攻击 - 魔物防御)
有效魔法伤害 = 队伍魔法攻击
弱点加成 = 魔物弱点命中时提供固定倍率
技能贡献 = 出战职业的自动技能提供的额外伤害、有效生命或有效防御
有效伤害 = 有效物理伤害 + 有效魔法伤害 + 伤害型技能贡献
预计回合数 = ceil(防守方生命 / 有效伤害)
```

若队伍击败魔物所需回合数小于或等于魔物击败队伍所需回合数，则讨伐成功。

## 轻量 RPG 规则

MVP 战斗保留确定性，不做命中、暴击和随机技能释放。RPG 元素只通过战前可读信息影响结算：

- 勇者有战士、法师、牧师等职业。
- 菜品可以强化基础属性，也可以强化对应职业的技能强度。
- 魔物可以有物理、魔法或圣光弱点。
- 弱点命中使用固定倍率，不使用随机概率。
- 职业技能自动结算，并在战斗日志中说明贡献。

首版弱点建议：

| 弱点 | 主要应对 | 设计目的 |
| --- | --- | --- |
| 物理弱点 | 战士或高攻击队伍 | 让攻击菜和战士有明确价值 |
| 魔法弱点 | 法师和奥术菜 | 解决高防魔物只堆攻击不划算的问题 |
| 圣光弱点 | 牧师和圣光菜 | 让牧师不只负责续航，也能参与克制 |

## 战斗展示

战斗可以先以简化日志展示：

- 出战勇者。
- 出战职业构成。
- 魔物属性。
- 魔物弱点和是否命中。
- 关键攻防差距。
- 职业技能贡献，如守护减少伤害、法术爆发增加伤害、治疗增加有效生命。
- 预计或实际回合数。
- 胜负结果。
- 获得奖励。

后续可增加演出：

- 勇者头像攻击序列。
- 魔物受击反馈。
- 关键数值跳字。
- 战利品掉落动画。

## 胜率提示

派遣前可以提供简化胜率或风险标签：

- 稳定胜利。
- 有风险。
- 极危险。
- 几乎无法取胜。

首版可由属性差距直接推导，不需要真实随机模拟。

## 扩展方向

后续可以加入：

- 更细职业克制。
- 魔物特性。
- 菜品临时增益。
- 战利品战斗被动。
- 多波次袭击。
- 主动或更复杂的勇者技能。

扩展必须保持战斗前可读，避免玩家无法理解结果。
