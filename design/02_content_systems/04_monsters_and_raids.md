# 魔物与袭击

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义魔物袭击内容、威胁等级、掉落和刷新规则。

## 袭击内容定位

魔物袭击是周期性目标和奖励来源。它应推动玩家思考当前勇者队伍是否足够强，以及下一步应该补哪类属性。

## 魔物类型方向

- 高攻魔物：考验队伍生命或防御。
- 高防魔物：考验队伍攻击。
- 高生命魔物：考验持续输出。
- 魔法弱点魔物：防御较高，鼓励玩家强化法师或制作奥术菜。
- 圣光弱点魔物：偏亡灵或腐化主题，鼓励玩家带牧师并强化技能强度。
- 群体魔物：后续可考验出战人数或范围能力。
- 精英魔物：提供更好掉落。

## 弱点设计

MVP 怪物弱点只保留三类，避免过早进入复杂元素克制。

| 弱点 | 推荐准备 | 典型魔物方向 |
| --- | --- | --- |
| 物理弱点 | 强化战士攻击，制作力量菜 | 哥布林、低甲野兽 |
| 魔法弱点 | 强化法师魔法攻击和技能强度，制作奥术菜 | 石壳兽、高防魔物 |
| 圣光弱点 | 强化牧师技能强度，制作圣光菜 | 幽灵、腐化狼群、亡灵 |

弱点信息必须在袭击预警中展示。玩家应能在倒计时期间根据弱点调整菜品和队伍，而不是战斗后才知道。

## 数据字段建议

| 字段 | 说明 |
| --- | --- |
| id | 魔物 ID |
| name_key | 名称文本键 |
| threat_level | 威胁等级 |
| hp | 生命 |
| attack | 攻击 |
| defense | 防御 |
| weakness_tags | 弱点标签，如 `weak_physical`、`weak_magic`、`weak_holy` |
| resistance_tags | 抗性标签，MVP 可暂不使用 |
| raid_timer_seconds | 袭击倒计时 |
| reward_table_id | 奖励表 ID |
| tags | 魔物标签 |

## 设计准则

- 魔物属性应反向提示玩家需要准备什么。
- 魔物弱点应提供清晰策略方向，而不是隐藏谜题。
- 袭击倒计时和奖励价值要匹配威胁等级。
- 首版不做过多特性，优先验证攻防公式和奖励闭环。
- 抗性系统首版仅保留数据字段预留，不进入 MVP 结算，避免惩罚玩家试错。
