# 勇者

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义勇者内容的职业、属性、档位和扩展规则。

## 勇者内容定位

勇者是餐厅吸引来的战斗单位。玩家通过菜品强化他们，再选择他们参与讨伐。

## MVP 职业方向

首版职业采用战法牧三角关系，保证玩家能根据魔物弱点做战前准备。

| 职业 | 战斗定位 | 主要属性 | 自动技能 | 适配食物 |
| --- | --- | --- | --- | --- |
| 战士 | 前排承伤与物理输出 | 生命 / 攻击 / 防御 | 守护：提高队伍有效防御或分摊伤害 | 力量 / 坚韧 |
| 法师 | 魔法爆发与高防克制 | 魔法攻击 / 技能强度 | 法术爆发：提供额外魔法伤害 | 奥术 |
| 牧师 | 续航、护盾与圣光克制 | 生命 / 技能强度 | 治疗或护盾：提高队伍有效生命 | 圣光 / 坚韧 |

后续可以扩展盾卫、游侠等细分职业，但 MVP 先不拆分，避免职业数量超过玩家理解成本。

## 数据字段建议

| 字段 | 说明 |
| --- | --- |
| id | 勇者 ID |
| name_key | 名称文本键 |
| rarity | 档位 |
| role | 职业定位 |
| base_hp | 基础生命 |
| base_attack | 基础攻击 |
| base_defense | 基础防御 |
| base_magic_attack | 基础魔法攻击 |
| base_skill_power | 基础技能强度 |
| auto_skill_id | 自动技能 ID |
| food_affinity_tags | 偏好菜品标签 |
| damage_tags | 伤害标签，如 `physical`、`magic`、`holy` |
| unlock_level | 所需餐厅等级 |

## 设计准则

- 勇者之间要有属性差异，避免只选总战力最高。
- 职业差异应在怪物弱点和战斗日志中可见。
- 高阶勇者可以有更高成长倍率，但首版不做手动技能。
- 勇者数量前期应少而清晰。
- 初始阵容建议至少包含战士、法师、牧师各 1 名，让玩家第一轮就理解职业用途。
