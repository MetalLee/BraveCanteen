# Boss 对策职业灵感

日期：2026-06-30  
来源：用户与 Codex 讨论  
关联：[[design/02_content_systems/03_adventurers|勇者]]、[[design/02_content_systems/04_monsters_and_raids|魔物与袭击]]、[[design/01_core_gameplay/06_auto_battle_system|自动战斗系统]]

## 原始问题

用户指出：法师和猎人如果负责属性克制、弱点破坏和行动限制，可能在复杂 Boss 战中变成战斗黑箱。因此需要设计一个特殊职业专门对抗 Boss。后续进一步确定：这个特殊职业命名为“游侠”，第二场战斗后解锁；原猎人方向重新设计为“弓箭手”，通过射击弱点让后续勇者小队伤害提高。

## 整理后的设计方向

新增特殊职业：游侠。

游侠不是最高输出，也不是最强防御，而是 Boss 战的信息和机制对策职业。它的职责是把 Boss 的隐藏机制提前转化为玩家能理解、能准备、能验证的战前信息。

## 核心功能

| 功能 | 说明 |
| --- | --- |
| 机制侦察 | 派遣前显示 Boss 狂暴阈值、护盾阶段、蓄力重击或属性弱点。 |
| 战术压制 | 降低一个 Boss 特性的强度，例如缩短护盾、降低狂暴、延后重击。 |
| 队伍协调 | 提示法师、弓箭手、盾兵、战士是否命中当前 Boss 的关键应对点。 |

## 弓箭手调整

原猎人职责过宽，容易同时承担破防、限制行动和弱点判断，增加战斗黑箱风险。调整后改为弓箭手，职责集中为：

- 射击敌人弱点。
- 让后续勇者小队对该目标伤害提高。
- 后续于 2026-07-01 再次调整：弓箭手不再是“标记后续增伤”，而是破坏部位造成出血。出血会让 Boss 每次攻击时扣除自身百分比生命，弓箭手再次攻击可造成可观伤害并叠加出血。

## 设计价值

- 降低 Boss 战黑箱感。
- 让复杂机制进入派遣前决策，而不是战后猜谜。
- 保留法师和弓箭手的职业特色，同时让玩家更容易理解它们何时有效。
- 给 Boss 战提供一个明确的阶段性解锁目标。

## 设计边界

- 游侠不能替代输出、承伤或部位破坏职业。
- 每个 Boss 首版只放 1 到 2 个核心机制。
- 游侠的提示必须在战斗日志中被验证。
- 游侠建议第二场战斗后解锁，避免玩家在首轮教学中过早接触 Boss 机制侦察。
