# 简单有趣的烹饪流程灵感

日期：2026-07-01  
来源：用户与 Codex 讨论  
关联：[[design/01_core_gameplay/02_restaurant_operations|餐厅经营]]、[[design/02_content_systems/02_recipes_and_dishes|食谱与菜品]]、[[design/03_experience/00_ui_ux|界面与交互]]、[[design/03_experience/04_feedback_and_game_feel|反馈与手感]]

## 原始问题

用户询问：如何让烹饪过程或者生成菜品简单有趣？

## 设计方向

烹饪不做复杂厨艺小游戏，而是做“少量选择生成战斗答案”。

首版采用三段式：

1. 选择底菜：汤、排、饭、沙拉、冷盘、轻食盒。
2. 选择主食材：肉块、蘑菇、小麦、青菜、元素食材。
3. 可选调味：香草、烈焰椒、霜盐等。

## 设计价值

- 简单：每次烹饪最多 2 个有效决策。
- 有趣：食材组合会改变战斗答案。
- 可预测：制作前显示属性和战斗预测变化。
- 有发现感：首次组合出新菜谱时记录进菜谱。

## 示例

| 组合 | 菜品 | 战斗用途 |
| --- | --- | --- |
| 排 + 肉块 | 烤肉排 | 基础攻击。 |
| 卷 + 青菜 + 肉块 | 青菜肉卷 | 战士攻击 + 精力。 |
| 汤 + 蘑菇 + 小麦 | 蘑菇盾锅 | 防御，多段攻击对策。 |
| 冷盘 + 青菜 + 霜盐 | 霜盐冷盘 | 法师冰属性爆发 + 精力。 |
| 轻食盒 + 青菜 + 香草 | 香草轻食盒 | 弓箭手部位破坏与出血准备。 |

## UI 注意

- 提供按职业推荐、按 Boss 缺口推荐、只看可制作、一键复做。
- 制作前显示“给谁吃、解决什么问题、预测如何变化”。
- 普通菜反馈快，首次发现新菜谱反馈稍重。
